# Google Maps Transit Partner Submission Process

## Overview

This document outlines the submission process for integrating Kocaeli public transit data into Google Maps through the Google Transit Partner program. Since this project is maintained by an independent solo developer, specific requirements must be met to publish the data.

## 1. Authorization Requirement

**CRITICAL:** Google requires explicit permission from the transit agency to publish data on their behalf. A solo developer cannot submit transit data without this authorization.

To proceed with the Google Maps Transit Partner program, you must obtain one of the following from the **Kocaeli Metropolitan Municipality**:
- A formal **Data License Agreement**.
- An **Official Letter of Authorization** stating that you are permitted to act as a data provider or technical contact for their transit data.

*Without this authorization, the GTFS data generated by this pipeline cannot be published to Google Maps.*

## 2. Data Quality Standards

Before submission, the GTFS data must meet Google's strict quality requirements:

### Static GTFS
- **Hosting**: The `gtfs.zip` file must be hosted on a stable, publicly accessible URL.
- **Validation**: The feed must pass the MobilityData GTFS Validator with no critical errors.
- **Freshness**: The feed should be updated regularly to reflect schedule changes.

### GTFS-Realtime
- **Hosting**: The `gtfs-rt.pb` file must be hosted on a stable, publicly accessible URL.
- **Matching IDs**: Trip, Route, and Stop IDs in the Realtime feed must exactly match the IDs defined in the Static GTFS feed.
- **Frequency**: The RT feed must be updated frequently (typically every 15-60 seconds, though 5 minutes is acceptable for this PoC phase).

## 3. Submission Steps

Once authorization is secured and data quality is validated:
1. Go to the [Google Transit Partner Dashboard](https://transitpartners.google.com/).
2. Sign in with a Google account.
3. Submit a request to add a new transit agency, providing the authorization documentation.
4. Add the Static GTFS URL (e.g., `https://<username>.github.io/<repo>/gtfs.zip`).
5. Add the GTFS-Realtime URL (e.g., `https://<username>.github.io/<repo>/gtfs-rt.pb`).
6. Complete the validation process initiated by the Google Transit team.
